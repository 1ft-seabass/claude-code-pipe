**修正した問題**:

#### 問題の原因
v0.7.0 で `sender.js` を options 形式に変更したが、`api.js` 側の呼び出しが個別引数形式のまま残っていた。

```javascript
// 問題のあった呼び出し（api.js）
sendToSession(sessionId, prompt, workingDirectory, allowedTools, ...)

// 実際のシグネチャ（sender.js）
function sendToSession(sessionId, prompt, options = {})
```

**結果**: `options` に文字列が渡され、`cwd` が `undefined` になり、セッションが claude-code-pipe 自身のディレクトリで起動していた。

#### 修正内容

**変更ファイル**:
- `src/api.js`: startNewSession/sendToSession を options 形式に修正、projectPath 追加
- `src/sender.js`: managedProcesses に projectPath を保存
- `src/canceller.js`: cancel-initiated イベントに projectPath を追加

**検証済み Webhook ペイロード**:
